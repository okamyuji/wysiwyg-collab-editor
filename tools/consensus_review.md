# Consensus Review プロトコル バージョン1.0

本書は Agent を用いたレビューの収束を保証するための実施手順である。

## 1. 受理基準

レビュアー(Agent)からの指摘は、以下の条件をすべて満たした場合のみ受理する。

1. 指摘が `QUALITY_RUBRIC.md` の判定項目に該当している
2. 指摘の重要度が `QUALITY_RUBRIC.md` 第7章の重要度定義に従っている
3. 同一の指摘(対象ファイル、対象行、根拠ルーブリック項目が一致)が N 体中 K 体以上の Agent で同時に出現している(多数決)

本プロトコルでは N=3、K=2(3体中2体以上の合意)を採用する。

## 2. 実施手順

| 順序 | 操作                                                                                     |
| ---- | ---------------------------------------------------------------------------------------- |
| 1    | 対象文書を明示し、`STYLE_GUIDE.md` と `QUALITY_RUBRIC.md` の参照を Agent へ含める        |
| 2    | 同じプロンプトで3体の Agent を並列に走らせる                                             |
| 3    | 各 Agent の指摘リストを集約し、(ファイル, 行範囲±2行, ルーブリック項目) のキーで突合する |
| 4    | 2体以上で合意した指摘を「正当な指摘」として採用する                                      |
| 5    | 単独指摘は「規約改訂候補」としてバックログに記録し、即時修正はしない                     |
| 6    | 正当な指摘をすべて修正する                                                               |
| 7    | 修正後、同じプロンプトで再度3体の Agent を走らせる                                       |
| 8    | 正当な指摘が0件であれば収束と判定する                                                    |
| 9    | 0件でない場合は手順4に戻る                                                               |

## 3. プロンプトの固定

レビュアーへ与えるプロンプトは以下に固定する。Agent 側の主観を排除し、ルーブリックに紐付けて指摘するよう要求する。

```
あなたは技術文書のレビュアーである。`docs/_quality/STYLE_GUIDE.md` (バージョン1.0) と
`docs/_quality/QUALITY_RUBRIC.md` (バージョン1.0) を品質基準として用いる。

対象ファイル: <PATH>

評価軸: QUALITY_RUBRIC.md に列挙された判定項目のみを用いる。判定項目に該当しない
主観的指摘は出さない。指摘ごとに必ず QUALITY_RUBRIC の項目番号(例「3. ADR充足表」)
を引用する。

フォーマット: - [Critical/High/Medium/Low][ファイル名:行番号][ルーブリック項目]内容

最低件数の要求はない。違反がなければ「違反なし」と返してよい。
```

## 4. 収束条件の保証

本プロトコルが収束する根拠は次の通りである。

1. プロンプトを固定するため、同じ Agent の主観差を排除できる
2. ルーブリックは外部に明文化されているため、Agent の判定基準が一致する
3. 多数決(K-of-N)により、Agent ごとの揺らぎを除外できる
4. ルーブリックを変更しない限り、同じ文書の判定結果は不変化する
5. ルーブリックを変更した場合は規約バージョンを上げ、追従改修した文書を再判定する

## 5. 改訂候補の扱い

単独指摘で受理されなかった項目は `docs/_quality/IMPROVEMENT_BACKLOG.md` に記録する。次バージョンの規約改訂時に検討する。
