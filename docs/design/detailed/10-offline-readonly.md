# 10オフライン閲覧

## スコープ

ServiceWorker、CacheStorage、`cache_etag` 算出、権限剥奪/共有リンク失効/文書削除の波及、ログアウト時のキャッシュ無効化を集約する。本MVPはオフライン編集を扱わない(FR-09第4項)。

## 契約

### キャッシュ範囲

| 対象     | 上限                |
| -------- | ------------------- |
| 文書     | 5件(最終アクセス順) |
| 画像     | 50枚                |
| UIシェル | 最新版1つ           |

ゲスト(`anyone` 型共有リンク)はキャッシュ対象外(`Cache-Control: no-store` でレスポンス)。restricted型共有リンクで認証済み利用者がアクセスする場合は通常認証フローと同等扱いでキャッシュ可。

### `cache_etag` 算出式

文書テーブルに列を持たず、サーバーが応答時に算出する派生値である。

```text
cache_etag = HMAC-SHA-256(
  key     = CACHE_ETAG_SECRET,
  message = documents.current_snapshot_version || ":" ||
            documents.metadata_updated_at      || ":" ||
            permission_level                   || ":" ||
            share_link_id  -- 認証済みはNULL
)
```

`CACHE_ETAG_SECRET` は `secret_versions` で管理する。`permission_level` を含めることで権限剥奪後のキャッシュは正しく無効化される(剥奪前後でETag不一致となる)。

### ネットワーク復帰時の整合性確認

| サーバー応答                                  | クライアント挙動                           |
| --------------------------------------------- | ------------------------------------------ |
| 200 +同じETag                                 | キャッシュ継続                             |
| 200 +異なるETag                               | 最新スナップショット再取得                 |
| 403 + `X-Cache-Invalidate: document/<doc_id>` | 当該文書キャッシュ即時削除、権限なし画面へ |
| 410                                           | 当該文書キャッシュ削除、一覧画面へ         |

### ログアウト時のキャッシュ無効化

```text
1. POST /api/auth/logout 成功時、クライアントはServiceWorkerへキャッシュクリア指示
2. ServiceWorkerは当該利用者の全文書キャッシュ、画像キャッシュ、ゲストCookieを削除
3. UIシェルは残してよい(再ログイン後の初期表示を速くするため)
4. ログアウト失敗時は再試行を促し、キャッシュ削除は完了まで継続
```

### ServiceWorker更新

| 項目                 | 値                                                     |
| -------------------- | ------------------------------------------------------ |
| ライブラリ           | Workbox                                                |
| 戦略                 | `skipWaiting` + `clientsClaim` 併用                    |
| `updateViaCache`     | `'none'`                                               |
| precacheManifest世代 | ビルドハッシュで差替、古い世代は新世代起動時に自動削除 |

### 共有リンク失効と権限剥奪の波及

| イベント       | 即時通知経路                                                               | オフライン時のフォロー                                                          |
| -------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| 共有リンク失効 | 当該リンク発行ゲストへCloseコード4410で通知、クライアントはキャッシュ削除  | 次回オンライン時の `cache_etag` 問い合わせでHTTP 410を受け削除                  |
| 権限剥奪       | 当該文書へのWebSocketをCloseコード4403で切断、クライアントはキャッシュ削除 | 次回オンライン時の `cache_etag` 問い合わせで403+`X-Cache-Invalidate` を受け削除 |
| 文書削除       | WebSocketで `force_reload`、クライアントはキャッシュ削除                   | 次回オンライン時のHTTP 410を受け削除                                            |

### エラーコード

| コード    | HTTP | 意味                                                       |
| --------- | ---- | ---------------------------------------------------------- |
| `OFF-001` | 403  | キャッシュ復帰時の権限剥奪検出                             |
| `OFF-002` | 410  | キャッシュ復帰時の文書削除検出                             |
| `OFF-003` | 0    | (クライアントローカル)キャッシュ上限超過で古いエントリ削除 |
