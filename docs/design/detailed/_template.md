# 詳細設計章テンプレート

このファイルは新しい詳細設計章を作るときの雛形である。実章では本コメント節を削除する。

## スコープ

この章が責任を持つ契約の範囲を1段落で書く。基本設計の対応節と、関連ADRをリスト化する。

## 契約

実装に必要な決定値を集約する。許される節は以下のとおりである。

### DDL (該当する場合)

```sql
CREATE TABLE example (...);
```

CHECK制約、トリガ、インデックスはすべて本節に置く。

### API契約(該当する場合)

| メソッド | パス           | リクエスト          | レスポンス       | エラーコード                       |
| -------- | -------------- | ------------------- | ---------------- | ---------------------------------- |
| POST     | `/api/example` | `{ field: string }` | `{ id: string }` | (本章接頭辞)-001、(本章接頭辞)-002 |

### CAS手順(該当する場合)

```sql
UPDATE example SET status='X', optimistic_version=optimistic_version+1
WHERE id=$id AND optimistic_version=$expected AND status='Y'
```

二段CASの場合は手順1から手順Nまでを順序付けて書く。

### アルゴリズムパラメータ(該当する場合)

| 項目                | 値    |
| ------------------- | ----- |
| 例: Argon2id memory | 19MiB |

### 定数(該当する場合)

| キー            | 値      | 単位 |
| --------------- | ------- | ---- |
| 例: SESSION_TTL | 1209600 | 秒   |

定数は最終的に `20-constants-registry.md` に一元化する。本章では定義+参照リンクで足りる。

### エラーコード(該当する場合)

エラーコードは `19-error-code-registry.md` に一元化する。本章では発生条件のみ書き、コード文字列はレジストリを参照する。
