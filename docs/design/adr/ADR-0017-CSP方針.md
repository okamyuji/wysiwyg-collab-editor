# ADR-0017 Content-Security-Policy方針

## ステータス

Accepted (2026-06-22)

## 文脈

本MVPはWYSIWYGエディタを含むSPAであり、XSS耐性を高める必要がある。CSPの厳格度はnonceのみ許可、`'unsafe-inline'` 全禁止が最も強い。レポート専用モードや段階導入は脆弱性窓を残す。

## 検討した選択肢

### 選択肢A: nonce方式の厳格CSP(本案)

script-srcとstyle-srcで `'self'` + nonceのみ許可、`'unsafe-inline'` 禁止。Quillの動的スタイルにはnonce付与する

### 選択肢B: hash方式

インラインスクリプトのSHA-256ハッシュをCSPに列挙。動的生成スクリプトには適用困難で、SPAのビルド変更ごとにCSP更新が必要となるため不採用

### 選択肢C: report-only段階導入

違反レポートを受けるのみで実効防御がない。本MVPの脆弱性窓を埋めるには不十分のため不採用

## 決定

選択肢Aを採用する。具体仕様は要件定義書および基本設計書の該当節に置き、本ADRは決定の所在と根拠を保持する。

## 根拠

採用根拠は以下のとおりである。

1. XSS防御の最も強い水準を初期から取り、後退ではなく前進方向で運用する。
2. Quill 2系はnonce付与の拡張ポイントを持ち、本案と両立する。
3. ペネトレーションテスト(NFR-04 14項)で厳格度を継続的に検証できる。
4. report-only移行は脆弱性窓を残すため、初期から強制モードで運用する。

## 影響

### ポジティブな影響

1. 主要設計決定のMECEな整理に寄与する。
2. 要件定義書および基本設計書とのトレーサビリティが明確になる。
3. 当該分野の確定値を1箇所に集約することで、将来の差し替えADR起票時の起点が明確になる。

### ネガティブな影響

1. 詳細仕様を要件定義書と基本設計書に委ねるため、ADR単体での自己完結性は限定的である。
2. 関連文書の更新時には本ADRも見直す必要がある。

### リスクと緩和策

1. Quillのインラインスタイル付与とnonce整合に開発初期の困難が予想される。緩和策として、Quill初期化時にnonceをstyle要素へ自動付与するラッパを実装する。
2. CDN経由のサードパーティスクリプト(分析ツールなど)導入時にCSP違反になる可能性がある。緩和策として、追加ドメインを許可リストに加える前にセキュリティ担当の承認を必須化する。
3. report-only緩和への将来圧力に対しては、差し替えADRを起票する手順で対応する。

## 自己レビュー記録

### レビュー観点1 (ADR-0017): nonceスコープの寿命

nonceはリクエストごとに発行し、レスポンス内で使い回す。複数リクエストへの再利用は禁ずる。

### レビュー観点2 (ADR-0017): CSP違反監視

Reporting APIで違反レポートを収集し、運用ダッシュボードに集約する。

### レビュー観点3 (ADR-0017): 差し替えトリガ

新しいフレームワーク採用時にCSP記述が困難となった場合は差し替えADRを起票する。
