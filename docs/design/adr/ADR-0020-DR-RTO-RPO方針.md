# ADR-0020 災害復旧の目標値

## ステータス

Accepted (2026-06-23)

## 文脈

本MVPの利用規模(同時編集者30名、全体5000接続)と業務上の停止許容時間を踏まえDR目標を確定させる。

## 検討した選択肢

### 選択肢A: RPO 24時間 / RTO 4時間(本案)

本MVPの規模で達成可能な水準

### 選択肢B: RPO 1時間 / RTO 30分

ストリーミングレプリカと自動フェイルオーバーが必要で運用コスト増

### 選択肢C: RPO 7日 / RTO 24時間

業務継続性が損なわれるため不採用

## 決定

選択肢Aを採用する。具体仕様は要件定義書および基本設計書の該当節に置き、本ADRは決定の所在と根拠を保持する。

### 確定値

| 指標        | 値                                             |
| ----------- | ---------------------------------------------- |
| RPO         | 24時間                                         |
| RTO         | 4時間                                          |
| 訓練頻度    | 半年に1回のリストア訓練、四半期に1回の机上演習 |
| BIAレビュー | 年1回                                          |

## 根拠

採用根拠は以下のとおりである。

1. 業務時間外の片肺運転で本目標を達成できる。
2. ストリーミングレプリカの導入は将来検討する。
3. 本MVP段階の利用規模では本目標が過不足ない水準である。
4. ビジネス影響評価(BIA)で1日の機能停止が業務に与える影響を評価し、本目標を妥当と判断した。

## 影響

### ポジティブな影響

1. 主要設計決定のMECEな整理に寄与する。
2. 要件定義書および基本設計書とのトレーサビリティが明確になる。
3. 当該分野の確定値を1箇所に集約することで、将来の差し替えADR起票時の起点が明確になる。

### ネガティブな影響

1. 詳細仕様を要件定義書と基本設計書に委ねるため、ADR単体での自己完結性は限定的である。
2. 関連文書の更新時には本ADRも見直す必要がある。

### リスクと緩和策

1. DR目標を満たすための予備サイト構築コストに対しては、マネージドサービス活用で抑制する。
2. ストレージ拡張に伴うバックアップ時間延長を継続監視する。
3. 実際の障害復旧時間が目標を上回る可能性に対しては、四半期ごとに机上演習を実施する。
