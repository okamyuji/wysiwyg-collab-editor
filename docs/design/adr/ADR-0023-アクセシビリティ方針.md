# ADR-0023 アクセシビリティの水準

## ステータス

Accepted (2026-06-25)

## 文脈

本MVPのアクセシビリティ準拠水準を確定させる。

## 検討した選択肢

### 選択肢A: WCAG 2.1レベルAA(本案)

業務システムの一般的な水準

### 選択肢B: WCAG 2.1レベルA

視覚的なコントラスト要件が不足するため不採用

### 選択肢C: WCAG 2.1レベルAAA

全文書一律でAAA達成は実装コスト過大のため不採用

## 決定

選択肢Aを採用する。具体仕様は要件定義書および基本設計書の該当節に置き、本ADRは決定の所在と根拠を保持する。

### 確定値

| 項目           | 値                                               |
| -------------- | ------------------------------------------------ |
| 準拠水準       | WCAG 2.1レベルAA                                 |
| ARIA仕様       | WAI-ARIA 1.2                                     |
| コントラスト比 | 4.5対1以上                                       |
| キーボード操作 | 主要操作はキーボードのみで完結                   |
| 色覚多様性     | 色のみで情報を伝達しない(ラベル併記)             |
| 自動テスト     | axe-coreによる自動評価をPlaywright E2Eに組み込む |
| 専門業者評価   | 年1回                                            |

## 根拠

採用根拠は以下のとおりである。

1. 公的調達基準の多くがAA相当を求めている。
2. Quill 2系のARIA属性とキーボード操作がAA水準を実装可能である。
3. WAI-ARIA 1.2のラベルとロールを採用する。
4. 色覚多様性に配慮し、プレゼンスの色だけでなく利用者名を併記する(要件定義書NFR-08)。

## 影響

### ポジティブな影響

1. 主要設計決定のMECEな整理に寄与する。
2. 要件定義書および基本設計書とのトレーサビリティが明確になる。
3. 当該分野の確定値を1箇所に集約することで、将来の差し替えADR起票時の起点が明確になる。

### ネガティブな影響

1. 詳細仕様を要件定義書と基本設計書に委ねるため、ADR単体での自己完結性は限定的である。
2. 関連文書の更新時には本ADRも見直す必要がある。

### リスクと緩和策

1. 装飾ツールバーのアイコンのみでは伝達不足の懸念があり、ツールチップでテキスト併記する。
2. スクリーンリーダー対応のテストが手動になり工数が増える。年1回の専門業者による評価を実施する。
3. 将来AAA達成圧力がある場合は、差し替えADRを起票する。
