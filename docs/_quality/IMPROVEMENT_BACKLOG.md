# 改善バックログ バージョン1.5

本書はConsensus Reviewで単独指摘となり受理されなかった項目、および将来の規約改訂候補を記録する。受理基準は決定論的に以下のとおりとする。

| 指摘の性質                          | 受理判定                                   | 適用先                           |
| ----------------------------------- | ------------------------------------------ | -------------------------------- |
| K-of-N=2-of-3合意あり               | 即時受理(本revisionで修正)                 | 該当ドキュメント                 |
| 単独指摘+技術妥当性特例カテゴリ該当 | 即時受理(本revisionで修正)、規約に観点追加 | 該当ドキュメント+ QUALITY_RUBRIC |
| 単独指摘+特例カテゴリ非該当         | 本revisionでは受理せず、本書に記録         | 本書のみ                         |
| 文体・好みの単独指摘                | 棄却                                       | 本書記録なし                     |

特例カテゴリ該当の判定はQUALITY_RUBRIC序文の6カテゴリ(データ整合性破綻、セキュリティ破綻、サービス不能リスク、MVP境界クロス衝突、文書間決定的矛盾、契約vs実装不一致)で行い、レビュアーが該当カテゴリを引用したものは原則受理する。次回のSTYLE_GUIDEまたはQUALITY_RUBRIC改訂時に検討する。
